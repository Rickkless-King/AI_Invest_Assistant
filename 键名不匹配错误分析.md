# 键名不匹配错误分析

## 问题现象

LLM输出显示所有股票数据都是0：
```
市值为0美元，最新成交价为0, 开盘最高/最低/开盘均为0，前收为0
```

---

## 根本原因：字典键名不匹配

### 错误示意图

```python
# 第1步：fetch_company_profile_with_fallback 返回
{
    '名称': 'NVIDIA',          # ✅ 中文键
    '行业': 'Semiconductors',  # ✅ 中文键
    '市值(百万美元)': 3500000   # ✅ 中文键（注意有括号）
}
        ↓
# 第2步：merge_all_data 尝试获取
profile.get('Industry', 'N/A')           # ❌ 找不到！返回 'N/A'
profile.get('MarketCapitalization', 0)   # ❌ 找不到！返回 0
```

---

## 详细对比

### 问题1: profile 键名不匹配

| 返回的键（实际） | 你使用的键（错误） | 结果 |
|------------------|-------------------|------|
| `'名称'` | `"名称"` | ✅ 正确 |
| `'行业'` | `'Industry'` | ❌ 返回 'N/A' |
| `'ipo时间'` | `'IPODate'` | ❌ 返回 'N/A' |
| `'市值(百万美元)'` | `'MarketCapitalization'` | ❌ 返回 0 |
| `'官网'` | `'weburl'` | ❌ 返回 'N/A' |
| `'描述'` | `'Description'` | ❌ 返回 'N/A' |

### 问题2: quote 键名不匹配

| 返回的键（实际） | 你使用的键（错误） | 结果 |
|------------------|-------------------|------|
| `"最新成交价"` | `'05. price'` | ❌ 返回 0 |
| `"当日最高价"` | `'03. high'` | ❌ 返回 0 |
| `"当日最低价"` | `'04. low'` | ❌ 返回 0 |
| `"当日开盘价"` | `'02. open'` | ❌ 返回 0 |
| `"前一个交易日的收盘价"` | `'08. previous close'` | ❌ 返回 0 |
| `"上述数据的更新时间"` | `'07. latest trading day'` | ❌ 返回 'N/A' |

---

## 为什么会犯这个错误？

### 错误1：混淆了API原始键名和转换后的键名

**你的思维过程可能是：**
```python
# Alpha Vantage API 原始返回：
{
    '05. price': 135.50,    # ← 这是API的原始键名
    '03. high': 138.20
}

# 但你已经在 fetch_real_time_data_with_fallback 中转换了：
return {
    "最新成交价": float(quote.get('05. price', 0)),  # 转换成中文键
    "当日最高价": float(quote.get('03. high', 0))
}
```

**在 `merge_all_data` 中应该使用转换后的中文键，而不是API原始键！**

### 错误2：没有注意括号

```python
# 返回的键名：
'市值(百万美元)'  # ← 有括号！

# 你使用的：
profile.get('市值', 0)  # ← 没有括号！匹配失败
```

## 正确的代码对比

### ❌ 错误版本

```python
@chain
def merge_all_data(x):
    profile = x["profile"]
    quote = x["quote"]

    return {
        # ❌ 使用了错误的键名
        '行业': profile.get('Industry', 'N/A'),           # 找不到！
        '市值': profile.get('MarketCapitalization', 0),   # 找不到！
        "最新成交价": quote.get('05. price', 0),          # 找不到！
    }
```

### ✅ 正确版本

```python
@chain
def merge_all_data(x):
    profile = x["profile"]
    quote = x["quote"]

    return {
        # ✅ 使用正确的中文键
        "行业": profile.get("行业", "N/A"),                    # ✅ 能找到
        "市值": profile.get("市值(百万美元)", "N/A"),          # ✅ 注意括号
        "最新成交价": quote.get("最新成交价", "N/A"),          # ✅ 使用转换后的中文键
    }
```

## 关键教训

### 1. 保持键名一致性

**原则：一旦转换成中文键，后续都用中文键**

```python
# Step 1: API返回 → 转换成中文
def fetch_data():
    raw = api.get_data()
    return {
        "名称": raw.get("name"),  # 转换
        "行业": raw.get("industry")
    }

# Step 2: 使用转换后的中文键
def merge_data(x):
    data = x["data"]
    return {
        "name": data.get("名称"),  # ✅ 使用转换后的键
        # NOT: data.get("name")   # ❌ 这是API原始键
    }
```

### 2. 注意特殊字符

```python
'市值(百万美元)'  # ← 括号是键名的一部分！
'市值'            # ← 这是不同的键
```

### 3. 使用打印调试

不确定时，先打印：
```python
print("可用的键:", list(profile.keys()))
```
